# Project Structure Reference

Detailed file-by-file documentation for the Redrob Candidate Discovery Engine repository.

## Root Files

| File | Purpose |
|------|---------|
| `main.py` | CLI entrypoint with `precompute`, `rank`, and `eda` subcommands |
| `rank.py` | Thin wrapper for competition `reproduce_command` |
| `validate_submission.py` | Official CSV format validator (copied from competition bundle) |
| `submission_metadata.yaml` | Portal metadata template — fill before submission |
| `requirements.txt` | Python dependencies with version bounds |
| `pytest.ini` | Pytest configuration and custom markers |
| `LICENSE` | MIT license |
| `.gitignore` | Excludes venv, large data, logs, caches |

## `config/`

| File | Purpose |
|------|---------|
| `jd_profile.yaml` | Structured hiring profile: must-have skills, title lists, career keywords, disqualifiers, behavioral thresholds, JD text for embedding |
| `ranking_weights.yaml` | Hybrid scorer weights, coarse filter thresholds, score range mapping |

## `data/raw/`

Competition bundle (not committed to git due to size):

| File | Size | Purpose |
|------|------|---------|
| `candidates.jsonl` | ~465 MB | 100,000 candidate records |
| `candidate_schema.json` | 9 KB | JSON Schema for candidate records |
| `job_description.docx` | 40 KB | Target role JD |
| `submission_spec.docx` | 43 KB | Format rules and evaluation criteria |
| `sample_submission.csv` | 9 KB | Format reference (deliberately bad ranking) |
| `sample_candidates.json` | 300 KB | First 50 candidates for inspection |
| `validate_submission.py` | 5 KB | Format validator |

## `data/processed/`

Generated by `python main.py precompute` (not committed to git):

| File | Purpose |
|------|---------|
| `candidate_embeddings.npy` | L2-normalized embeddings for all 100K candidates (~146 MB) |
| `jd_embedding.npy` | JD text embedding vector |
| `candidate_ids.json` | Ordered list of candidate IDs matching embedding rows |
| `bm25_index.pkl` | Tokenized career corpus + BM25 index |
| `career_corpus.json` | Career description texts used for BM25 |
| `honeypot_ids.json` | Detected honeypot candidate IDs |

## `src/`

| Module | Responsibility |
|--------|----------------|
| `utils.py` | Paths, YAML loading, DOCX parsing, JSONL iteration, logging setup |
| `preprocessing.py` | Canonical text builder, honeypot/keyword-stuffer/consulting/title-chaser detection, title scoring |
| `semantic_matching.py` | Sentence-transformer embeddings, BM25 index, precompute/load artifacts, batch similarity |
| `feature_engineering.py` | Trusted skill match, YoE/education/location, career narrative, quality penalties |
| `behavioral_scoring.py` | Redrob signal modifier (response rate, activity, notice period, etc.) |
| `ranking_engine.py` | Two-stage filter, hybrid scoring, reasoning generation, CSV writer |
| `evaluation.py` | Validator subprocess wrapper, honeypot rate check, submission summary |

## `scripts/`

| Script | Purpose |
|--------|---------|
| `run_eda.py` | Full dataset audit → `outputs/eda_report.json` |
| `parse_jd.py` | Optional: refresh `jd_profile.yaml` from `job_description.docx` |

## `notebooks/`

| Notebook | Purpose |
|----------|---------|
| `01_dataset_audit.ipynb` | Interactive EDA wrapper around `run_eda.py` |

## `tests/`

| Test File | Coverage |
|-----------|----------|
| `test_validator.py` | Official validator accepts sample submission |
| `test_honeypot.py` | Honeypot and keyword-stuffer detection |
| `test_ranking.py` | Title scoring, YoE, trusted skills |
| `test_benchmark.py` | Full 100K ranking under 5 min, honeypot gate, validator |

## `outputs/`

| File | Purpose |
|------|---------|
| `submission.csv` | **Final competition submission** |
| `eda_report.json` | Dataset statistics and trap counts |
| `benchmark.json` | Runtime, top-10 IDs, honeypot rate, validation status |
| `*.log` | Precompute/rank logs (gitignored) |

## `docs/`

| File | Purpose |
|------|---------|
| `TECHNICAL_REPORT.md` | Full methodology and validation report |
| `PRESENTATION.md` | 10-slide deck content with speaker notes |
| `PROJECT_STRUCTURE.md` | This file |

## Data Flow

```
candidates.jsonl
    │
    ├─► precompute ──► embeddings.npy, bm25_index.pkl, honeypot_ids.json
    │
    └─► rank ──► preprocessing ──► feature_engineering ──► behavioral_scoring
                      │                    │                      │
                      └──── semantic_matching (batch dot product + BM25)
                                           │
                                    ranking_engine
                                           │
                                    submission.csv
```

## Module Dependencies

```
main.py
 ├── semantic_matching
 ├── ranking_engine
 │    ├── preprocessing
 │    ├── feature_engineering
 │    ├── behavioral_scoring
 │    └── semantic_matching
 ├── evaluation
 └── scripts/run_eda
      └── preprocessing
```
